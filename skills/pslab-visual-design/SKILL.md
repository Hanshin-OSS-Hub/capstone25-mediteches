# PS Lab 시각 디자인 가이드

pslab-hci-ux(상호작용 원칙)·pslab-ui-patterns(코드 패턴)과 **보완 관계**.
이 스킬은 "어떻게 보여야 하는가"에 집중한다.

> 참고한 디자인 리더십: **Linear** (2026 UI refresh), **Stripe** (Design System),
> **Vercel** (Developer Dashboard), **5of10** (Dashboard Best Practices 2025)

---

## 0. 핵심 철학

> "Don't compete for attention you haven't earned." — Linear 2026
> "The best dashboards become invisible—users see insights, not design." — 5of10

1. **주요 콘텐츠가 주인공**. 네비게이션·보조 요소는 뒤로 물러난다.
2. **구조는 느끼되 보이지 않게** (Structure should be felt, not seen). 테두리를 줄이고 여백·배경 차이로 구분한다.
3. **5초 테스트**: 화면을 5초 보여주고 감췄을 때, 핵심 지표와 상태를 말할 수 있어야 한다. 못 하면 단순화한다.

---

## 1. 카드를 기본으로 쓰지 않는다

카드는 **독립된 객체** (상품, 프로젝트 등)에만 쓴다. 같은 유형을 나열할 때 카드를 반복하면 시각적 노이즈다.

| 상황 | ❌ 카드 | ✅ 대안 |
|------|--------|--------|
| 동일 유형 나열 (과목, 알림, 멤버, 상점 아이템) | 카드 그리드/캐러셀 | `divide-y` 리스트 행 |
| 통계 3~5개 | 카드 그리드 | 히어로 내 인라인 스탯 + 구분선 |
| 시간순 이벤트 (공지, 로그) | 카드 스택 | 타임라인 (세로선 + 도트) |

### 대안 패턴

- **리스트 행**: `divide-y` + 상태 도트 + hover 배경. 테두리 없이 여백으로 구분.
- **인라인 스탯**: `flex divide-x` — 숫자 + 라벨을 구분선으로 배치. 카드 없음.
- **타임라인**: `absolute w-px bg-gray-200` 세로선 + 원형 도트 + 우측 콘텐츠.
- **섹션**: `border-b` 한 줄 또는 배경색 차이로 영역 분리.

---

## 2. 3계층 색상 시스템

Stripe·Vercel이 공유하는 색상 원칙. 색은 **장식이 아니라 의미 전달 도구**.

### 계층 1: 시맨틱 (상태)

| 의미 | 색 | 용도 |
|------|---|------|
| 성공 | 에메랄드 `#10b981` | 완료, 정상, 확인 |
| 위험 | 레드 `#ef4444` | 삭제, 에러, 휴강 |
| 경고 | 앰버 `#f59e0b` | 주의, 개발중 |
| 정보 | 블루 `#3b82f6` | 링크, 안내 |

이 4색은 **의미 전용**. 다른 용도로 빌려쓰지 않는다.

### 계층 2: 강조 (CTA·포인트)

한 페이지에 **강조색 1개만**. PS Lab 기본 강조: 인디고 `#6366f1`.
그라데이션은 히어로·헤더에만 허용.

### 계층 3: 중성 (UI 기본)

| 요소 | 라이트 | 다크 |
|------|--------|------|
| 본문 | `gray-900` | `zinc-100` |
| 보조 | `gray-500` | `zinc-400` |
| 비활성 | `gray-400` | `zinc-500` |
| 배경 | `white` | `zinc-900` |
| 카드/영역 | `gray-50` | `zinc-800/40` |
| 테두리 | `gray-200` | `zinc-700` |

### 금기

- 한 화면에 **강한 색 3가지 이상** 동시 사용 금지.
- 아이콘은 `text-gray-400` 기본. 강조 시에만 색을 넣는다 (Linear 원칙).
- 카드마다 다른 border-color 금지 (노이즈).

## 3. 구조는 느끼되 보이지 않게 — 테두리·여백

Linear 2026 리프레시의 핵심: 테두리와 구분선을 줄이고, 남은 것은 **둥글게·약하게**.

### 테두리 원칙

- **같은 영역 내 항목**: `divide-y` 또는 `space-y`. 각 항목에 border를 두르지 않는다.
- **다른 영역 사이**: `mb-6` 이상 간격 또는 배경색 차이.
- **필요한 테두리**: `border-gray-100 dark:border-zinc-800` — 약한 대비. `border-gray-200`도 너무 강할 수 있음.
- **모서리**: Vercel 기준 `rounded-lg` (6px). 히어로·큰 영역만 `rounded-2xl`.

### 여백 리듬 (4px 그리드 — Vercel)

Spacing: `4, 8, 12, 16, 24, 32, 48px` — Tailwind의 `1, 2, 3, 4, 6, 8, 12`.
리스트 행: `py-3.5` (14px) — 높이 32~36px 유지.

### 그림자

- `shadow-sm` 하나만. `shadow-md` 이상 과도.
- Vercel 스타일: 카드와 배경의 밝기 차이가 **2~4%**만. 그림자보다 배경색 차이로 계층 표현.

## 4. 히어로 배너

페이지 최상단에 **하나의 그림**으로 핵심 지표·제목·상태를 보여준다. 히어로 안에 인라인 스탯을 넣으면 별도 카드 그리드가 불필요.

```
┌─ 그라데이션 (rounded-2xl) ───────────────────┐
│  아이콘 + 제목              [컴팩트 정보 칩]  │
│  부제 (요약 한 줄)                             │
│                                                │
│  숫자1  │  숫자2  │  숫자3                      │
│  라벨1  │  라벨2  │  라벨3                      │
│                                                │
│  ▓▓▓▓▓▓▓░░░ 프로그레스 바 (선택)               │
└────────────────────────────────────────────────┘
```

- 히어로는 **컴팩트하게**. 불필요하면 인라인 스탯 수를 줄이거나 부제에 통합.
- 배경: `bg-gradient-to-br from-{color}-600 to-{color2}-500` / 다크 `from-{color}-900 to-{color2}-900`
- 오버레이: `bg-[radial-gradient(ellipse_at_top_right,rgba(255,255,255,0.15),transparent_70%)]`

---

## 5. 탭 — 언더라인 스타일

Linear 2026: 탭 바를 더 컴팩트하게, 아이콘·텍스트 크기를 줄였다.

```tsx
<div className="flex border-b border-gray-200 dark:border-zinc-700">
  <button className={`px-4 py-3 text-sm font-medium relative ${
    isActive ? "text-indigo-600 dark:text-indigo-400"
             : "text-gray-500 hover:text-gray-700"
  }`}>
    {label}
    {isActive && <span className="absolute bottom-0 left-2 right-2 h-0.5 rounded-full bg-indigo-600" />}
  </button>
</div>
```

- **아이콘 넣지 않는다**. 텍스트 라벨만으로 충분.
- 뱃지: `text-[10px] font-bold` 숫자만. 배경 불필요.
- pill/box 탭 금지 — 공간 과다, 시각적으로 무겁다.

## 6. 타이포그래피 위계

| 레벨 | 스타일 | 용도 |
|------|--------|------|
| 히어로 숫자 | `text-3xl font-bold text-white tabular-nums` | 핵심 지표 |
| 페이지 제목 | `text-xl~2xl font-bold text-gray-900 dark:text-zinc-100` | 최상위 |
| 섹션 제목 | `text-sm font-bold` | 카테고리·그룹 |
| 항목 이름 | `text-sm font-medium` | 리스트 행 |
| 보조 텍스트 | `text-xs text-gray-400 dark:text-zinc-500` | 날짜·메타 |
| 미니 라벨 | `text-[10px]` | 뱃지·범례 |

- 숫자: `tabular-nums`. 비교 맥락 필수 (vs 지난 기간, 목표 대비).
- 최소 폰트: `14px` (= `text-sm`). 그 아래는 메타·뱃지에만.
- 행간: `leading-snug` (제목), `leading-relaxed` (본문).

---

## 7. 호버·전환 — 절제 (Vercel 원칙)

Vercel: **"Communicates competence through restraint."** 전환은 **200ms ease**, 절대 bounce 하지 않는다.

- 리스트 행: `hover:bg-gray-50/80 dark:hover:bg-zinc-800/40`. 미세한 변화.
- 숨긴 액션: `opacity-0 group-hover:opacity-100 transition-opacity`.
- 전환: `transition-colors duration-200`. `transition-all`은 레이아웃 변화 시에만.
- `animate-pulse`: 로딩 스켈레톤·상태 도트에만. 과용 금지.
- bounce·spring 애니메이션 금지.

---

## 8. 우측 상단 비우기

PS Lab은 **플로팅 프로필이 우측 상단**에 위치한다. 본문 영역 우측 상단에 정보·버튼을 두면 프로필과 겹치거나 시선이 경쟁한다.

- 인라인 정보 (잔액, 상태 칩): **왼쪽 정렬** 또는 히어로 내부.
- 페이지 액션 버튼 (새로고침, 외부 링크): **제목 옆(왼쪽)**.
- sticky 요소: `justify-start`. `justify-end` 금지.

---

## 9. 다크 모드 — Vercel 스타일

Vercel: 다크 모드는 기능이 아니라 기본. 개발자는 어두운 환경에서 일한다.

| 원칙 | 값 |
|------|---|
| 배경 | warm charcoal `zinc-900` — 순수 검정 금지 |
| 카드/영역 | 배경보다 **2~4% 밝게만** (`zinc-800/40~60`) |
| 텍스트 | off-white `zinc-100` — 순수 흰색 금지 |
| 강조색 | 채도 15~20% 낮춤 (`{color}-400`, 라이트의 600 대비) |
| 테두리 | `zinc-700~800` — 라이트보다 약하게 |

### 다크 모드 가독성

다크 모드에서는 대비가 부족해 **구분선·텍스트가 배경에 묻히기 쉽다**. 항상 다크 모드 화면을 직접 확인하라.

- `divide-y`: 라이트 `divide-gray-100`은 다크에서 안 보인다 → `dark:divide-zinc-700` 이상.
- 보조 텍스트: `text-gray-400`은 다크 배경에서 안 보인다 → `dark:text-zinc-400` (500 아닌 400).
- 비활성 텍스트도 `dark:text-zinc-500` 이상. `zinc-600` 이하는 배경에 묻힌다.
- 구분선·테두리는 다크에서 **한 단계 밝게**: 라이트 `gray-100` → 다크 `zinc-700`.

### 필수 체크

새 요소를 추가할 때 `dark:` 클래스를 **동시에** 작성한다. 나중에 넣으면 누락된다.
**다크 모드에서 직접 눈으로 확인**하고 넘어간다.

## 10. F 패턴 · 정보 구조

아이트래킹 연구(5of10): 사용자는 **F 패턴**으로 스캔한다.

- **좌상단**: 가장 중요한 KPI 또는 제목.
- **상단 행**: 핵심 지표 3~5개 (히어로 인라인 스탯).
- **중앙**: 트렌드·리스트.
- **하단**: 상세 테이블·설정.

### 지표 표시 원칙 (5of10)

- 숫자만 놓지 말고 **비교 맥락**을 제공: vs 지난 기간, vs 목표.
- 한 화면에 **핵심 지표 5~7개 이하**. 넘으면 탭·접기로 분리.
- 갱신 주기가 다른 지표를 섞을 때 "마지막 업데이트" 표시.

## 11. 레퍼런스 대상

| 회사 | 배울 점 |
|------|---------|
| **Linear** | 보조 요소 후퇴, 테두리 최소화, 아이콘 축소, warm gray |
| **Stripe** | 시맨틱 토큰, 8px 그리드, 보라=CTA만, 신뢰=명확함 |
| **Vercel** | 거의 단색 팔레트, 2~4% 밝기 차 계층, 200ms ease |
| **Notion** | 넉넉한 여백, 콘텐츠 우선, 미니멀 아이콘 |

**참고하지 않을 것**: LMS 관리자 패널, ERP 대시보드, 공대생 프로젝트 — 기능은 많지만 감각이 없다.

## 자가 점검

1. **5초 테스트** 통과하는가?
2. 화면에 **테두리 사각형이 5개 넘는가?** → 줄여라.
3. **같은 색 뱃지가 3개 이상** 반복되는가? → 색을 빼거나 도트로.
4. 빈 공간이 **답답한가?** → 패딩·마진 늘려라.
5. 스크린샷이 **"공학적"**으로 보이는가? → 테두리 줄이고 여백 늘려라.
6. 전환이 **bounce·spring** 하는가? → `200ms ease`로 교체.
7. **우측 상단**에 무언가 있는가? → 왼쪽으로 옮겨라.

## 체크리스트 (리디자인 시)

1. 카드 → 리스트 행·타임라인·인라인 스탯으로 바꿀 수 있는가?
2. 색상이 3계층(시맨틱·강조·중성) 안에 있는가?
3. 히어로 배너가 컴팩트하게 핵심을 보여주는가?
4. 탭이 언더라인 스타일인가?
5. 타이포 위계가 3단계 이상 명확한가?
6. 테두리 대신 여백·배경 차이로 구분하는가?
7. hover 액션이 평소 숨겨져 있는가?
8. 우측 상단이 비어 있는가?
9. 다크 모드에서 `dark:` 클래스가 빠진 곳이 없는가?
10. Linear/Stripe와 비교했을 때 과한 부분이 있는가?
