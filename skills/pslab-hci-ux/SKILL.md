# HCI / UX 가이드라인 (PS Lab)

에이전트는 **프론트엔드를 작성하거나 수정하기 직전**에 이 스킬을 읽고 아래 원칙을 적용한다. 기존 코드를 건드릴 때도 위반 사항이 보이면 점진적으로 개선한다.

---

## 1. 피드백 (Feedback)

모든 사용자 액션에는 **즉각적이고 눈에 보이는 응답**이 있어야 한다.

- **버튼** — 클릭 후 로딩 상태(`disabled` + 스피너)를 즉시 표시. 완료/실패 시 **토스트·인라인 메시지**로 결과 알림.
- **토글/체크** — 낙관적 업데이트(optimistic update): UI를 먼저 바꾸고 API 응답을 기다린다. 실패 시 되돌린다.
- **폼 제출** — 성공 메시지 or 에러 하이라이트. 에러는 **해당 필드 근처**(인라인)에 표시, 화면 상단에만 뿌리지 않는다.
- **빈 상태(Empty state)** — 데이터가 0건일 때 "아직 없습니다" + 생성 안내(아이콘 + 짧은 문구 + CTA 버튼).
- **삭제·위험 동작** — `ConfirmModal`(프로젝트 표준)로 확인. `window.confirm` 금지.

## 2. 가시성 (Visibility of System Status)

- **로딩** — 스켈레톤 또는 `Loader2 animate-spin` + 텍스트. 빈 화면 없이 로딩 상태를 알린다.
- **진행도** — 여러 단계(위자드·다중 폼)는 **스텝 인디케이터** 또는 진행 바.
- **현재 위치** — 사이드바 활성 탭, 페이지 제목·브레드크럼 등으로 사용자가 어디에 있는지 알 수 있게.
- **데이터 신선도** — 새로고침 버튼 + 마지막 로드 시각, 자동 리패치 주기가 긴 경우 "오래된 데이터일 수 있습니다" 안내.

## 3. 인지 부하 줄이기 (Cognitive Load)

- **밀러의 법칙** — 한 화면에 선택지·정보 덩어리는 **5~9개 이하**. 넘으면 카테고리·탭·접기로 나눈다.
- **청킹** — 긴 리스트·테이블은 그룹 제목(섹션 헤더)·구분선으로 시각적 블록 분리.
- **점진적 노출(Progressive Disclosure)** — 고급 옵션은 접기(`details/summary` 또는 "더 보기") 뒤에 둔다. 초기 화면은 핵심 정보만.
- **기본값** — 폼 필드에 합리적인 기본값(`placeholder` ≠ 기본값; 실제 value로). 사용자가 바꿀 일이 적으면 미리 채워 둔다.
- **라벨 > 아이콘** — 아이콘만으로 의미가 모호하면 **텍스트 라벨을 병행**한다.

## 4. 일관성 (Consistency)

- **컬러 시맨틱** — 프로젝트 내 의미 통일: 빨강=위험·삭제, 에메랄드=성공·확인, 블루=정보·링크, 앰버=경고.
- **버튼 패턴** — 주요 CTA: `bg-{color}-600 text-white rounded-lg px-4 py-2 text-sm font-semibold`. 보조: `border + text-gray-700 hover:bg-gray-50`.
- **간격·모서리** — 카드 `rounded-2xl`, 내부 섹션 `rounded-xl`, 버튼 `rounded-lg`. 패딩 `p-4 ~ p-6`.
- **타이포** — 페이지 제목 `text-2xl font-bold`, 섹션 제목 `text-lg font-bold`, 본문 `text-sm`.
- **아이콘** — Lucide React만 사용. 크기는 UI 맥락에 맞춰 `w-4 h-4`(인라인) ~ `w-6 h-6`(헤더).

## 5. 접근성 (Accessibility)

- **키보드** — 모든 인터랙티브 요소(`button`, `a`, `input`, `select`)는 키보드 포커스 가능. `tabIndex` 누락 금지.
- **포커스 링** — `focus:ring-2 focus:ring-{color}-200 outline-none`. 포커스 표시를 제거하지 않는다.
- **`aria-label`** — 아이콘만 있는 버튼에 필수 (`title`만으로 충분하지 않음).
- **색 대비** — 회색 텍스트는 `text-gray-500` 이상 (400 이하 금지). 배경 위에서 WCAG AA 이상.
- **모달** — `role="dialog"`, `aria-modal="true"`, Esc로 닫기, 배경 클릭 닫기. `ConfirmModal` 참고.

## 6. 에러 방지 & 복구 (Error Prevention)

- **입력 제약** — `type="number"` + `min`/`max`, `maxLength`, `required` 등 HTML 수준에서 1차 차단.
- **비활성화** — 필수 필드가 비었으면 제출 버튼 `disabled:opacity-50`.
- **되돌리기** — 삭제 후 "되돌리기" 토스트(가능한 경우) 또는 최소한 `ConfirmModal`로 2차 확인.
- **에러 메시지** — "오류 발생" 같은 범용 문구 대신 **무엇이 잘못됐고 어떻게 고치는지** 안내.

## 7. 반응형 (Responsive)

- **모바일 우선** — `max-w-5xl mx-auto` 등 컨테이너. 테이블은 `overflow-x-auto`.
- **그리드** — 카드 그리드: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`.
- **터치** — 터치 대상은 최소 `44×44px`(= `w-11 h-11` 또는 `px-3 py-2.5` 이상).
- **가로 스크롤** — 카드 행이 넘칠 때 `snap-x snap-mandatory overflow-x-auto` + `shrink-0 snap-start` 아이템.

## 8. 정보 구조 (Information Architecture)

- **F 패턴** — 가장 중요한 정보(제목·CTA)를 왼쪽 상단에 배치.
- **근접성(Proximity)** — 관련 항목은 가까이, 무관한 항목은 공간·선으로 분리.
- **시각적 위계** — 크기·굵기·색으로 중요도 차등 (예: 금액은 `font-bold tabular-nums`, 보조 정보는 `text-xs text-gray-400`).

## 9. 마이크로인터랙션 (Microinteraction)

- **호버** — 카드·행: `hover:bg-gray-50`, 버튼: `hover:bg-{color}-700`.
- **트랜지션** — 상태 변화는 `transition-colors` 또는 `transition-all duration-200`.
- **토스트** — 4초 자동 소멸 + 닫기 버튼. 여러 건 중첩 가능하면 스택.

## 체크리스트 (새 UI 작성 시)

코드를 작성하기 전 또는 PR 전에 아래를 확인한다.

1. 모든 버튼·인풋에 **로딩·에러·성공 상태**가 있는가?
2. **빈 상태**(0건)에 안내 문구·CTA가 있는가?
3. 삭제·위험 동작은 **`ConfirmModal`** 을 쓰는가?
4. 폼 기본값이 합리적인가? 사용자가 매번 채워야 하는 칸이 많지 않은가?
5. **한 화면에 정보 덩어리가 9개를 넘으면** 탭·접기·그룹으로 나눴는가?
6. **아이콘만 있는 버튼**에 `aria-label`이 있는가?
7. 모바일에서 터치 대상이 너무 작지 않은가? 테이블이 가로로 잘리지 않는가?
8. 색 시맨틱이 프로젝트 규칙과 일치하는가 (빨강=위험, 에메랄드=확인…)?
9. 기존 페이지(회비·상점·게시판 등)와 **버튼·카드·간격 톤**이 통일되는가?
